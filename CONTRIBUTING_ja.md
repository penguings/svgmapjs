# SVGMap への貢献ガイドライン

SVGMap プロジェクトへの貢献を検討していただき、ありがとうございます。このプロジェクトをより良くするためのガイドラインを以下に示します。

## 開発の進め方

### ブランチ戦略
- 貢献（プルリクエスト）は `dev*` ブランチ（例: `dev2`）に対してのみ受け付けられます。
- `main` ブランチへの直接のプルリクエストは原則として受け付けていません。

### 環境構築
プロジェクトをローカルで実行するには、以下の手順が必要です。

```bash
# 依存関係のインストール
npm install

# Playwright（E2Eテスト用）のインストール
npx playwright install
sudo npx playwright install-deps
```

## コード規約

### フォーマッタ
プロジェクト全体で `prettier` を使用しています。コミット前にフォーマットを確認してください。
設定は `.prettierrc` に準拠し、インデントにはタブを使用します。

```bash
# フォーマットの実行（例）
npx prettier --write .
```

## テストの実行

変更を加えた後は、必ず以下のテストを実行して、既存の機能に影響がないことを確認してください。

### ユニットテスト
Jest を使用したロジックのテストです。

```bash
npm test
```

### E2E テスト
Playwright を使用したブラウザ上での動作確認テストです。

```bash
npm run e2e
```

## Issue の報告

バグを発見した場合や、機能の改善案がある場合は、GitHub の [Issue](https://github.com/svgmap/svgmapjs/issues) を作成してください。
再現手順や環境情報を詳細に記載いただけると、スムーズに対応できます。

## プルリクエストの送り方

大きな変更を加える場合は、事前に Issue を作成して変更内容を相談することをお勧めします。プロジェクトの方向性との一致を確認し、作業の重複を避けることができます。

1. このリポジトリをフォークします。
2. 新しい機能や修正のためのブランチを作成します（例: `feature/your-feature-name`）。
3. 変更を加え、テストがすべてパスすることを確認します。
4. `dev*` ブランチに対してプルリクエストを作成します。
5. 変更内容を簡潔に説明し、関連する Issue があれば明記してください。
